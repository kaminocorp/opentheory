## 1. Intent

Make the new integrity primitives usable from the product, built on the existing TanStack-Query reads/mutations + query-key invalidation pattern (no new client-state library). A user can now: assess a claim (or a checkpoint) and see the outcome; fork a competing line from a checkpoint, record checkpoints on it without touching the main line, and close it as a dead-end with its reasoning preserved; and see at a glance which claims are contested and which branches are abandoned.

## 3. Design choices worth recording

- **What "scope to a branch" means here.** Claims belong to threads, not branches, so branch selection scopes the **ledger line** ŌĆö the checkpoint timeline and new-checkpoint `branch_id` ŌĆö while the center claims panel stays thread-scoped. This is the model-accurate reading of the plan's "scopes the center/right panels"; forcing claims under branches would invent a relationship the backend doesn't have.
- **Client-side line filtering.** `listCheckpoints` already returns every checkpoint with its `branch_id`, so the timeline filters locally rather than calling `GET /branches/{id}` ŌĆö one query feeds both main line and every branch view, and switching lines is instant.
- **Validation history only where there's an endpoint.** Claims have `GET /claims/{id}/validations`, so they show full history. Checkpoints have no per-target list endpoint, so the checkpoint surface is record-only; the resulting `validate` checkpoint is the visible artifact.
- **Contradiction indicator is a client heuristic** (status `challenged`, or any `contradicts`/`failed` validation). The plan defers the authoritative derived signal to the `0.4.4` read model; this is the interim surface.

- No backend read-model enrichment: no project-level branch/validation counts, no server-derived claim signal, no per-branch checkpoint counts in the header (`0.4.4`).

- No merge UI (`merged` status never appears ŌĆö deferred, plan Decision #3).
- No artifact validation in the UI (no artifact write flow).
- Branch selection scopes the ledger line, not the claim set (see ┬¦3).
- No adversarial multi-agent review pass yet.
- Changelog `0.4.x` index entries not added here ŌĆö left until the phases are confirmed, consistent with how `0.3.x` landed.
